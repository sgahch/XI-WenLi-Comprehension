### RuoYi-Vue 项目运行指南

本指南将引导您完成 RuoYi-Vue 项目（v3.8.9）前后端的启动步骤。

**一、 环境准备**

在运行项目之前，请确保您的开发环境已安装以下软件：

1.  **JDK:** Java Development Kit, 版本 >= 1.8
2.  **Maven:** Java 项目管理工具, 版本 >= 3.0
3.  **Node.js:** JavaScript 运行环境, 版本 >= 12.0
4.  **Redis:** (非必须，但推荐) 用于缓存和会话管理。请确保 Redis 服务已启动并运行在默认端口 (6379)，或者根据需要修改后端配置文件中的 Redis 连接信息。
5.  **MySQL:** 数据库, 版本 >= 5.7 (或 8.0+)。

**二、 后端启动步骤**

1.  **数据库初始化:**
    *   找到项目根目录下的 `sql` 文件夹。
    *   执行 `quartz.sql` 文件，创建定时任务相关的表。
    *   执行 `ry_2023xxxx.sql` 文件 (请选择最新的版本)，创建项目业务相关的表。
2.  **修改数据库配置:**
    *   打开后端主模块 `ruoyi-admin` 下的配置文件：`ruoyi-admin/src/main/resources/application-druid.yml`。
    *   找到 `datasource` 部分，修改 `url`, `username`, 和 `password` 以匹配您的 MySQL 数据库连接信息。
3.  **修改 Redis 配置 (如果需要):**
    *   打开后端主模块 `ruoyi-admin` 下的配置文件：`ruoyi-admin/src/main/resources/application.yml`。
    *   找到 `redis` 部分，根据您的 Redis 服务器信息修改 `host`, `port`, `password` (如果有)。
4.  **编译并运行后端:**
    *   在项目 **根目录** (包含 `pom.xml` 文件的目录) 打开终端或命令行。
    *   **方式一 (使用 Maven 插件):**
        *   运行命令：`mvn clean package -Dmaven.test.skip=true` (如果第一次运行或有代码修改，先打包)
        *   运行命令：`mvn spring-boot:run -pl ruoyi-admin` (或者直接在 IDE 中找到 `ruoyi-admin` 模块下的 `RuoYiApplication.java` 文件并运行其 `main` 方法)
    *   **方式二 (运行 Jar 包):**
        *   先执行打包命令：`mvn clean package -Dmaven.test.skip=true`
        *   打包成功后，在 `ruoyi-admin/target` 目录下会生成 `ruoyi-admin.jar` 文件。
        *   运行命令：`java -jar ruoyi-admin/target/ruoyi-admin.jar`
    *   当看到类似 "... Tomcat started on port(s): 8080 (http) ..." 或 "... RuoYiApplication started ..." 的日志时，表示后端启动成功。默认端口是 `8080`。

**三、 前端启动步骤**

1.  **进入前端目录:**
    *   在终端或命令行中，切换到前端代码目录：`cd ruoyi-ui`
2.  **安装依赖:**
    *   运行命令：`npm install` (或者如果您使用 `yarn`, 运行 `yarn install`)
    *   这将下载并安装前端项目所需的所有依赖包。请耐心等待安装完成。
3.  **修改 API 地址 (如果后端端口不是 8080):**
    *   打开前端配置文件：`ruoyi-ui/.env.development`
    *   找到 `VUE_APP_BASE_API` 变量。
    *   默认值通常是 `/dev-api`，它会通过代理指向后端。检查 `ruoyi-ui/vue.config.js` 文件中的 `proxy` 设置，确保 `target` 指向您后端运行的实际地址和端口 (例如 `http://localhost:8080`)。如果后端端口修改了，请相应修改 `vue.config.js` 中的 `target`。
4.  **启动前端开发服务器:**
    *   运行命令：`npm run dev` (或者 `yarn dev`)
    *   当前端编译成功并启动开发服务器后，您会在终端看到类似 "... App running at: - Local: http://localhost:80 ..." 的信息。前端默认运行在 `80` 端口 (可能需要管理员权限) 或其他端口 (如 `81`, `8081`)，请留意终端输出的实际访问地址。

**四、 访问系统**

*   打开您的浏览器。
*   在地址栏输入前端服务运行的地址 (例如 `http://localhost:80` 或终端提示的其他地址)。
*   您应该能看到若依系统的登录页面。
*   使用默认用户名 `admin` 和密码 `admin123` 登录系统。
